## 核心目标
在Windows 11系统下，编写一个程序作为Openclaw的启动容器：

1. **彻底消除CMD窗口**：Openclaw启动及运行过程中，禁止任何黑色CMD控制台窗口弹出，仅保留启动器GUI窗口；
2. **Win11现代界面**：提供圆角、Mica/Acrylic效果、跟随系统暗黑/亮色模式；
3. **原样彩色输出**：内置带彩色输出的终端区域，实时捕获Openclaw及其子进程的标准输出，**完整保留原生ANSI彩色格式**（红色错误、绿色成功等），**日志持久化到文件，启动时截断保留1万行**；
4. **托盘状态可视化**：托盘使用🦞龙虾图标，根据Openclaw运行状态显示不同颜色（**灰色=停止，黄色=启动中（静态），绿色=运行中**）；
5. **全局热键控制**：支持 Ctrl+Shift+O 随时切换主窗口显示/隐藏（需管理员权限运行），**支持运行时通过UI配置自定义热键**；
6. **自动发现与显示**：启动时自动查找Openclaw可执行文件（支持`.exe`和`.cmd`）、配置文件、**以openclaw配置文件中的端口为准**（默认18789），并在主窗口和终端显示完整信息；
7. **PID状态监控**：监控Openclaw进程PID，进程退出时自动触发重启流程；
8. **两步启动确认**：先监控命令输出（含子批处理/子进程）判断启动完成，再执行URL健康检查确认；
9. **安全启动机制**：**仅PID监控触发的自动重启**失败时自动重试，最多2次，两次失败后停止；用户手动启动/重启操作不计入重试次数；
10. **托盘驻留控制**：点击最小化或关闭按钮均缩到托盘，**只能通过托盘右键菜单退出程序**；
11. **配置加载策略**：配置仅在启动器启动时加载，用户修改配置文件后需重启启动器生效；
12. **完整进程管理**：接管Openclaw的启动、监控、终止流程，确保无进程残留；
13. **命令执行可视化**：启动器自主执行的所有命令在终端显示准确命令行和完整输出（包括子进程/子批处理输出）；
14. **开机自启动冲突检测**：启用开机自启动时，检测系统中是否有其他 OpenClaw 相关的计划任务，避免冲突。

## 详细技术要求

### 1. 可执行文件兼容（Windows）

**支持的文件类型**：
- `openclaw.exe` - 可执行文件
- `openclaw.cmd` - Windows批处理脚本

**查找顺序**：
1. 配置文件中指定的路径
2. 当前目录
3. `%APPDATA%\.openclaw\` 目录
4. PATH 环境变量
5. 用户手动选择（弹出文件选择对话框）

### 2. 无窗口执行（核心）

**关键要求**：
- 通过管道重定向 stdout/stderr 进行捕获
- 兼容Openclaw启动子进程的场景：子进程同样无窗口运行
- 非阻塞启动，保证GUI终端和托盘功能正常响应

**实现要点**：
- 设置 `CREATE_NO_WINDOW` 标志
- 设置 `STARTF_USESHOWWINDOW` 并将 `wShowWindow` 设为 0 (SW_HIDE)
- 不使用 shell 执行（`shell=False`）

### 3. 自动发现流程（全程可视化）

**端口策略**：默认18789，若成功读取配置文件则使用配置端口覆盖

**首次运行行为**：
- 若配置文件不存在，启动器**自动创建**该文件
- 自动保存自动发现到的配置（openclaw路径、配置文件路径、端口等）
- 用户后续修改需重启启动器生效

```
启动器启动
    ↓
[步骤1: 查找可执行文件]
    ├── 配置中有路径 → 验证存在 → 显示路径
    ├── 当前目录 → 找到 → 保存并显示
    ├── %APPDATA%\.openclaw\ → 找到 → 保存并显示
    ├── PATH 环境变量 → 找到 → 保存并显示
    └── 都未找到 → 弹出文件选择对话框
    ↓
[步骤2: 查找配置文件]
    ├── openclaw所在目录\config.yaml (或 openclaw.json)
    ├── %USERPROFILE%\.openclaw\config.yaml (或 openclaw.json)
    └── 都未找到 → 终端显示警告，使用默认端口18789
    ↓
[步骤3: 读取网关端口]
    ├── 配置文件存在且解析成功 → 端口覆盖默认值18789
    └── 失败 → 显示"默认端口 18789"
    ↓
[步骤4: 保存配置]
    └── 自动保存到配置文件
```

### 4. 命令执行与输出捕获

**核心要求**：启动器自主执行的所有命令，必须在日志区域显示**准确的命令行**和**完整的命令输出**（包括子进程/子批处理输出）。

**覆盖范围**：
- 启动命令
- 重启命令
- 停止命令
- 任何其他启动器自主执行的命令

### 5. PID状态监控机制

**核心逻辑**：
- RUNNING状态下，后台线程每秒检查一次PID是否存在
- PID消失（进程退出）→ 自动触发重启流程（最多2次重试）
- 不主动进行HTTP健康检查，除非在启动确认阶段

### 6. 两步启动确认流程

**第一步：监控输出或健康检查判断启动完成（总超时不超过15秒）**
```
执行启动命令
    ↓
状态 = STARTING（图标黄色静态）
    ↓
实时监控stdout/stderr（含子批处理/子进程输出）
    ↓
检测启动完成标志（正则匹配 ready_pattern）
    ↓
[匹配成功] → 进入第二步
    ↓
[超时未匹配] → 并行进行健康检查
    ↓
健康检查通过 → 进入第二步
    ↓
健康检查失败 → 判定启动失败
```

**就绪模式 (ready_pattern)**：
- 默认正则: `listening|ready|Gateway started`
- 可在配置文件中自定义

**第二步：URL健康检查确认（最终验证，超时5秒）**
```
HTTP GET请求 health_check_url
    ↓
[HTTP 200 且响应包含 health_check_keyword] → 状态=RUNNING（图标绿色）
    ↓
[失败] → 判定启动失败
```

### 7. 启动重试机制（仅自动重启计数）

**关键规则**：**仅PID监控触发的自动重启**计入重试次数，用户手动点击"启动"/"重启"按钮**不计数**

```
用户点击"启动"或"重启"按钮:
    ↓
重置 try_count = 0
    ↓
执行两步启动确认流程
    ↓
[失败] → 状态=STOPPED，显示错误（不自动重试）

PID监控检测到进程退出（自动重启场景）:
    ↓
try_count += 1
    ↓
try_count > max_start_attempts ?
    ├── 是 → 状态 = STOPPED，显示"自动重启失败（已尝试N次）"
    └── 否 → 等待 retry_delay 秒后执行两步启动确认流程
```

### 8. 手动停止与重启

**手动停止**：
- 标记 `manual_stop = True`（防止误触发自动重启）
- 执行 `openclaw gateway stop` 命令（取消计划任务注册）
- 强制终止Openclaw进程树（`taskkill /T /F`）
- 状态 = STOPPED
- 重置 try_count = 0

**手动重启**：
- 重置 try_count = 0
- 停止当前进程
- 进入启动流程

### 9. 日志持久化

**日志文件**：`launcher.log`（与启动器同级目录或 `%APPDATA%\OpenClawLauncher\`）

**行为**：
- 所有终端输出同时写入日志文件
- 日志中去除ANSI颜色代码，保留纯文本内容
- **启动时截断**：保留最新的1万行
- 运行期间只追加写入
- 日志格式：`[2024-01-15 10:30:25] [INFO] 输出内容`
- 线程安全（使用锁）

### 10. 配置加载策略

**规则**：
- 配置仅在**启动器启动时**读取一次
- 运行期间修改配置文件不会实时生效
- 修改配置后需**重启启动器**才能加载新配置

### 11. 全局热键

**默认热键**: `Ctrl+Shift+O`

**功能**:
- 切换主窗口显示/隐藏
- 需要管理员权限

**运行时配置**:
- 点击"设热键"按钮进入配置模式
- 终端显示提示"按下你想要的组合键..."
- 按下组合键后自动保存到配置文件
- 按 ESC 取消配置
- 支持的修饰键: Ctrl, Alt, Shift, Win
- 支持的按键: A-Z, 0-9, F1-F24, 特殊键

**权限检查**:
- 启动时检查是否以管理员权限运行
- 若无权限，显示警告但程序继续运行（热键功能禁用）

### 12. 开机自启动

**实现方式**：Windows 任务计划程序 (schtasks)

**创建任务**：
```batch
schtasks /create /tn OpenClawLauncher /tr "\"path\to\exe\"" /sc onlogon /delay 0000:30 /rl limited /f
```

**可选：VBScript 包装器**（用于隐藏窗口）：
```vbscript
Set WshShell = CreateObject("WScript.Shell")
WshShell.Run "\"path\to\exe\"", 0, False
Set WshShell = Nothing
```

**冲突检测**：
- 启用开机自启动前，检查是否有其他 OpenClaw 相关的计划任务
- 若存在冲突任务，显示弹窗列出冲突任务名称
- 提供"打开计划任务管理器"按钮，引导用户手动删除冲突任务

**参数说明**：
- `/delay 0000:30` - 延迟30秒启动，避免开机时资源竞争
- `/rl limited` - 以普通用户权限运行

### 13. 最小化到托盘动画（可选）

**实现步骤**：
1. 捕获窗口截图
2. 创建分层窗口显示截图
3. 隐藏主窗口
4. 执行缩放+移动动画
5. 动画结束后销毁分层窗口

**动画参数**：
- 帧数: 8
- 帧间隔: 16ms
- 缩放: 从 100% 到 10%
- 透明度: 从 255 到 50
- 移动: 从窗口中心到托盘图标中心

## 核心逻辑流程

### 启动器完整启动流程
```
启动器启动
```

## 已知问题与解决方案

### 1. 托盘菜单动态更新
- **问题**：菜单显示后文字不更新
- **解决**：使用定时器在菜单显示期间定期更新菜单项

### 2. 按钮ID冲突
- **问题**：窗口按钮和托盘菜单使用相同ID导致处理混乱
- **解决**：使用不同的ID，在命令处理中分别处理

### 3. 智能滚动
- **问题**：用户查看历史日志时自动滚动干扰
- **解决**：监听滚动事件和选择事件，只在用户位于底部且未选择文本时自动滚动

### 4. 终端背景色问题
- **问题**：启动时终端显示灰色而非深色背景
- **解决**：确保终端控件直接设置背景色，不叠加其他控件

### 5. 开机自启动冲突
- **问题**：系统中可能存在其他 OpenClaw 相关的计划任务
- **解决**：启用前检测冲突任务，引导用户手动删除

## 注意事项（给开发者）

1. **无窗口执行**：必须正确设置进程创建标志，确保无CMD窗口弹出；
2. **ANSI颜色解析**：主动解析 `\033\[[0-9;]*m` 序列；
3. **进程树清理**：Windows下使用 `taskkill /T /F` 确保无残留；
4. **热键权限**：全局热键需要管理员权限运行；
5. **图标资源**：若缺失，动态生成纯色圆形图标；
6. **PID监控**：每秒检查一次，避免忙等待；
7. **日志截断**：启动时截断到1万行；
8. **端口优先级**：必须以openclaw配置文件中的端口为准；
9. **重试计数器**：仅PID监控触发的自动重启计数，用户手动操作重置计数器；
10. **手动停止不重启**：用户点击"停止"按钮后，进程退出不触发自动重启；
11. **配置加载**：只在启动时读取一次，运行期间不监听文件变化；
12. **Win11专用**：可充分利用Windows特有API，无需考虑跨平台兼容；
13. **冲突检测**：启用开机自启动前检查现有任务，避免重复或冲突；
14. **线程安全**：日志写入和终端更新需要线程安全处理。
